1.2 Architecture modulaire

Le backend est structuré en apps Django strictes :

  • apps/accounts/ — Auth, profils, rôles, follows, analytics partenaires
  • apps/feed/ — Posts, likes, comments, stories, media, sauvegardes, partages
  • apps/chat/ — Conversations, messages, WebSocket
  • apps/ar_filters/ — Filtres RA, manifest géolocalisé
  • apps/geo/ — Points d'intérêt, nearby discovery
  • apps/notifications/ — Notifications push, tokens FCM
  • config/ — Settings, URLs, ASGI, Celery

3. Ce qui reste à faire (aligné sur le cahier des charges)

3.1 Intégrations externes (dépendances)

DépendanceRôlePrioritéCe qu'il faut fournirImpact si absent
Cloudflare R2 Stockage objets S3-compatible pour les médias (photos, vidéos) et les assets des filtres AR. Remplace AWS S3 sans frais d'egress, critique pour les JOJ. P0 - AWS_ACCESS_KEY_ID : clé API R2
- AWS_SECRET_ACCESS_KEY : secret API R2
- AWS_S3_ENDPOINT_URL : endpoint S3-compatible (ex: https://<account_id>.r2.cloudflarestorage.com)
- AWS_STORAGE_BUCKET_NAME : nom du bucket (ex: discover-senegal-media) 		Bloquant : impossible d'uploader des médias ou des filtres AR. L'application ne peut pas créer de contenu.
Maadi AI Moteur IA pour deux fonctions clés : (1) recommandations personnalisées du feed selon le comportement et la localisation, (2) modération automatique du contenu avec score de confiance. P0 - MAADI_AI_URL : URL du service (ex: https://maadi.discover-senegal.sn/recommend)
- MAADI_AI_TIMEOUT : timeout d'appel en ms (recommandé: 300) 		Non bloquant : le feed bascule en mode dégradé (tri par popularité) et la modération automatique est désactivée. L'application reste utilisable mais moins pertinente et moins sûre.
Firebase Admin Service de notifications push (FCM) pour iOS et Android. Permet d'alerter les utilisateurs des likes, messages, nouveaux followers et événements JOJ. P0 - Fichier firebase-admin-credentials.json : clé de service générée depuis la console Firebase
- Variable FIREBASE_CREDENTIALS_PATH : chemin d'accès au fichier sur le serveur (ex: /etc/secrets/firebase-admin-credentials.json) 		Non bloquant : aucune notification push ne peut être envoyée. L'application reste fonctionnelle mais les utilisateurs ne reçoivent pas d'alertes en temps réel.
Google OAuth Login social Google pour réduire la friction d'inscription. P1 - GOOGLE_OAUTH_CLIENT_ID et GOOGLE_OAUTH_CLIENT_SECRET dans Google Cloud Console
- Configuration dans settings Django 		Non bloquant : l'inscription par email/password reste disponible.
Apple OAuth Login social Apple pour réduire la friction d'inscription. P1 - APPLE_OAUTH_SERVICE_ID, APPLE_OAUTH_KEY_ID, APPLE_OAUTH_TEAM_ID et APPLE_OAUTH_PRIVATE_KEY dans Apple Developer
- Configuration dans settings Django 		Non bloquant : l'inscription par email/password reste disponible.
Banuba/DeepAR SDK SDK tiers pour l'application de filtres RA sur caméra mobile (face tracking, OTA). P1 - Licence SDK Banuba ou DeepAR
- Effect packs uploadés sur R2 + CDN 		Non bloquant : les filtres RA statiques fonctionnent sans SDK dynamique.
Watermarking Tâche Celery pour ajouter le watermark "Discover Sénégal" sur les médias uploadés. P1 - Implémenter tâche Celery avec Pillow
- Intégrer au flow d'upload média 		Non bloquant : les médias sont uploadés sans watermark.
